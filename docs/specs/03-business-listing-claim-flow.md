## Executive Summary

The Business flow enables store owners and repair service providers to add new listings or claim existing ones. It uses a unified "lookup first" pattern that prevents duplicates and routes users to the appropriate flow (claim vs. add).

### Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Entry point | Single "Add or Claim" button | Reduces confusion, smart routing |
| Authentication | Guest submit â†’ Account for claim | Lower friction for submissions |
| Email verification | Required before review | Reduces spam, validates ownership |
| Moderation | Manual review (MVP) | Quality control for new directory |
| Form structure | 5 steps (detailed) | Complete listings from start |
| Business types | Store + Provider (separate forms) | Different data requirements |

## Security: Claim Verification Process

### Race Condition Protection

The claim approval system uses PostgreSQL advisory locks to ensure atomic operations:

1. **Advisory Lock**: When an admin approves a claim, the system acquires a transaction-level lock using `pg_try_advisory_xact_lock()`. This prevents two admins from approving different claims for the same business simultaneously.

2. **Row-Level Lock**: The target store/provider row is locked with `SELECT ... FOR UPDATE` before ownership is transferred.

3. **Automatic Rejection**: If a business is already claimed during the approval process, the claim is automatically rejected with a clear explanation.

```sql
-- Simplified flow in process_claim_approval()
1. Acquire advisory lock for the specific business
2. Check if business already has owner (with row lock)
3. If already owned â†’ Auto-reject claim with reason
4. If available â†’ Transfer ownership, reject other pending claims
5. Lock automatically releases at transaction end
```

### Email Domain Verification (Future Enhancement)

For Phase 2, implement email domain matching:
- Verify claimant's email domain matches the business's registered email domain
- Example: `john@acme-appliances.com` claiming `Acme Appliances Ltd`
- Auto-approve if domains match; manual review otherwise

### Rate Limiting

Claims are rate-limited to prevent abuse:

| Limit | Value | Scope |
|-------|-------|-------|
| Claims per user per day | 3 | Per authenticated user |
| Claims per business per user | 1 | Cannot re-claim same business |
| Verification attempts | 5 | Per claim, then locked |
